Features

Built for modern hardware

Every stage of the sync pipeline is parallelized, vectorized, and zero-copy. No single-threaded bottlenecks.

B3

BLAKE3 + SIMD

256-bit cryptographic hashing with AVX-512/AVX2/NEON auto-vectorization. 10-15x faster than MD5 with stronger security guarantees.

||

Parallel Everything

Rayon work-stealing thread pool across all stages: directory scanning, hashing, delta computation, and file application. Uses all your cores.

0c

Zero-Copy I/O

Memory-mapped files with madvise(SEQUENTIAL|HUGEPAGE), copy_file_range(2) kernel-space copies, fallocate(2) pre-allocation.

CDC

Content-Defined Chunking

FastCDC with Gear rolling hash. Insertions at byte 0 no longer invalidate every chunk. 95%+ hash preservation on prepend edits.

Zs

Zstd Compression

Facebook Zstandard replaces zlib: 10x faster compression at equivalent ratios for network transfers.

TLS

Native TLS 1.3

Memory-safe TLS via rustls. No OpenSSL dependency. Push and pull modes with certificate verification.

Mc

Manifest Cache

Two-tier manifest: tiny TOC for dir-mtime checks + zstd-compressed bincode manifest. Skip destination scanning entirely on repeat syncs.

At

Atomic + Crash-Safe

Write to temp, then rename(2). fallocate(2) pre-allocation. No partial files ever. Nanosecond timestamp precision via utimensat(2).

Sh

Security Hardened

Path traversal guards, decompression bomb limits, setuid stripping, streaming writes. Rust memory safety eliminates buffer overflows.

Xa

Extended Attributes

Preserve xattrs with --xattrs/-X. Sparse file support, append mode, reflinks (CoW), one-file-system boundaries.

Tm

--trust-mtime

Ultra-fast no-change detection via directory mtimes. Sub-25ms at 100K files. Safe for CI/CD and deployment pipelines.

Pp

Push & Pull

Binary protocol over TCP with TLS 1.3. Push files to a remote server or pull from one. No SSH tunnel required.

Internals

Architecture

Two-phase pipeline: sequential classification, then parallel I/O. Files sorted by inode for optimal disk access.

  Scanner (adaptive: sequential turbo / parallel BFS)
      |
      v
  FilterEngine (exclude / include / size / glob)
      |
      v
  Phase 1: Sequential Decision (O(1) HashMap lookups)
      |
      |-- New file -------> Applier::copy_new()  (copy_file_range / atomic)
      |
      |-- Skip -----------> mtime+size / --size-only / manifest
      |
      |-- Update ---------> Hasher (BLAKE3 + mmap)
      |                        |
      |                        v
      |                    CdcChunker (FastCDC Gear hash)
      |                        |
      |                        v
      |                    DeltaEngine (hash-map match)
      |                        |
      |                        v
      |                    Applier::apply() (atomic write)
      |
      +-- Delete ---------> --delete / --max-delete guard

  Phase 2: Parallel I/O (rayon work-stealing pool)
      All per-file work runs on a scoped rayon pool.
      Files sorted by inode for optimal disk I/O order.
scanner.rs

FastCDC content-defined chunking (Gear hash, normalized)

Delta computation: diff two FileManifests (index + hash-map)

Glob matching, rate limiter, itemize changes, backup helpers
sync_engine.rs
Core orchestrator: two-phase scan then decide then apply pipeline
net/
Binary protocol, async TCP server/client, TLS 1.3 transport
Trust

Security

Defense in depth. Every layer validated.

🔐

BLAKE3 (256-bit)

Cryptographically secure hash. No known collisions or preimage attacks. Immune to length-extension.

🔒

TLS 1.3 (rustls)

Memory-safe TLS. No OpenSSL. Certificate verification against system CAs. Forward secrecy.

🛡

Path Traversal Guards

Server and client reject ".." components and absolute paths. Server blocks writes to system directories.

💣

Decompression Limits

Zstd messages capped at 256 MiB decompressed. Oversized payloads rejected.

⚙

setuid/setgid Stripping

Non-root users automatically strip setuid and setgid bits from synced files.

📝

Atomic Writes

Write to temp then rename(2). A crash never leaves a corrupt file at the destination.
