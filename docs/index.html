rsync reimagined in Rust. Parallel. Zero-copy. Unstoppable.

Capabilities

Engineered for speed at every layer

Every component of resync was designed from scratch to exploit modern hardware -- SIMD hashing, work-stealing parallelism, zero-copy I/O, and lock-free data paths.

B3

BLAKE3 Hashing

SIMD-accelerated BLAKE3 with AVX-512 support. Cryptographically strong, unlike rsync's broken MD5. Processes data at memory bandwidth on modern CPUs.

||

Rayon Parallelism

Work-stealing thread pool saturates all CPU cores automatically. File scanning, hashing, and delta computation run fully parallel -- rsync is single-threaded.

0C

Zero-Copy I/O

Memory-mapped files via mmap with madvise hints. Data moves kernel-to-kernel with copy_file_range(2). No userspace buffer copies on supported kernels.

CDC

Content-Defined Chunking

FastCDC with Gear rolling hash for shift-resistant boundaries. Insertions and deletions only affect local chunks, minimizing transfer deltas.

Zs

Zstd Compression

Facebook's Zstandard replaces zlib: 10x faster compression at equivalent ratios. Adaptive compression levels based on available bandwidth.

TLS

Native TLS 1.3

Built on rustls -- a memory-safe TLS implementation. No OpenSSL dependency. TLS 1.3 only: faster handshakes, forward secrecy by default.

At

Atomic Writes

Crash-safe by design. Writes go to a temp file, then atomically renamed via rename(2). fallocate(2) pre-allocation eliminates fragmentation. No partial files, ever.

HP

Huge Pages

MADV_HUGEPAGE for transparent huge page support on large files. Reduces TLB misses by orders of magnitude for multi-gigabyte datasets.

ns

Nanosecond Precision

utimensat(2) preserves file timestamps with nanosecond granularity. Sparse file support, append mode, and one-file-system boundaries handled natively.

rsync vs resync

Parallelism Single-threaded Rayon work-stealing (all cores)
I/O Strategy read(2) + write(2) mmap + copy_file_range(2)
Chunking Fixed-size blocks FastCDC (Gear rolling hash)

Write Safety Partial file risk Atomic temp + rename
Pre-allocation None fallocate(2), zero fragmentation
Timestamp Precision Microsecond Nanosecond (utimensat)
Large File Optimization None MADV_HUGEPAGE, transparent huge pages
Network Protocol Custom text-based Binary protocol, push mode
CLI Compatibility -- 100% rsync flag compatible

Usage

Quick start

100% compatible with rsync flags. If you know rsync, you already know resync.

Basic Sync
# Mirror a directory (archive mode)
resync -avz /src/ /dst/

# Sync with delete (remove extraneous files)
resync -avz --delete /src/ /dst/

# Dry run -- preview changes first
resync -avzn /src/ /dst/
Remote Sync
# Push to remote host
resync -avz /data/ user@host:/backup/

# Pull from remote host
resync -avz user@host:/data/ /local/

# Custom SSH port
resync -avz -e "ssh -p 2222" /src/ host:/dst/
Advanced Options
# Append mode for growing log files
resync --append-verify logs/ /backup/logs/

# Stay on one filesystem
resync -avz --one-file-system / /mnt/bkup/

# Include/exclude patterns
resync -avz --exclude "*.tmp" \
  --include "*.rs" src/ dst/
Network Push Mode
# Start resync daemon
resync --daemon --port 8730

# Push with TLS encryption
resync -avz --tls /data/ \
  resync://host:8730/module/

# Bandwidth limit (10 MB/s)
resync -avz --bwlimit=10m /src/ /dst/
